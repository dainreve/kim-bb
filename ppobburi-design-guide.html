PPOBBURI Design System

Design Guidelines & Standards
Version 1.0 | 2026년 5월
01. 개요
뽀뿌리의 디자인 시스템은 전통한복의 우아함과 현대적 심플함을 조화시킨 미니멀리즘 철학을 따릅니다.

"Beyond the BLAS, Embrace Tradition"

이 가이드는 일관된 브랜드 경험을 제공하기 위해 색상, 타이포그래피, 레이아웃, 컴포넌트 등의 규격을 정의합니다.
✓ 핵심 원칙: 전통과 현대의 조화, 명확한 계층 구조, 사용자 중심 디자인
02. 색상 시스템
Primary Colors
Black (메인)

White (배경)

Secondary Colors
Light Gray
#F5F5F5

#999999 보조 텍스트, 힌트 정보 계층
03. 타이포그래피
폰트 선택
뽀뿌리는 전통성을 표현하기 위해 세리프 폰트와 현대성을 위해 산스-세리프 폰트를 혼합 사용합니다.
한글 (메인)
Noto Serif KR - 전통성 표현
Noto Sans KR - 현대성 표현
영문
Georgia - Serif (고급 느낌)
Helvetica - Sans Serif (현대적)
타이포그래피 규격
용도 크기 굵기 줄높이
페이지 제목 36-48px 700 (Bold) 1.2
섹션 제목 24-28px 600 (Semi Bold) 1.3
본문 14-16px 400 (Regular) 1.6
보조 텍스트 12-13px 400 (Regular) 1.5
✓ 원칙: 한글/한자 병기로 전통성 강조, 명확한 계층 구조
04. 레이아웃
그리드 시스템
12-컬럼 그리드 시스템으로 일관된 여백과 정렬을 유지합니다.
항목 규격
컨테이너 너비 1200px (데스크톱), 100% (모바일)
컬럼 개수 12 컬럼
갭 20px
기본 마진 20px ~ 40px
카드 간격 24px
화이트스페이스
충분한 여백을 사용하여 상품 이미지와 텍스트를 명확히 구분합니다. 최소 16px 이상의 패딩을 유지합니다.
✓ 원칙: 넉넉한 여백으로 미니멀리즘 구현, 가독성 우선
05. 주요 컴포넌트
버튼
Primary Button
배경: #000
텍스트: #FFF
Secondary Button
배경: Transparent
테두리: #000
카드
상품 카드는 흰색 배경에 검은색 테두리로 구성되며, 호버 시 그림자로 강조합니다.
⚠ 주의: 카드 간격은 최소 20px 이상으로 유지하여 각 상품을 명확히 구분하세요.
네비게이션
상단 네비게이션은 검은색 텍스트를 사용하며, 현재 페이지는 언더라인으로 표시합니다.
요소 규격
메뉴 텍스트 크기 14-16px
메뉴 간격 24-32px
활성 상태 표시 언더라인 (2px) 또는 배경색
06. 포토그래피
뽀뿌리는 한복의 아름다움을 극대화하는 포토그래피를 중시합니다.
가이드라인
  • 자연광 또는 따뜻한 조명으로 한복 색감을 살립니다
  • 여유 있는 구성으로 한복의 디테일을 강조합니다
  • 배경은 미니멀하게 (흰색, 베이지, 회색)
  • 모델은 한복을 정중하게 입은 자세를 표현합니다
  • 여러 각도의 사진으로 상품을 다각도로 보여줍니다
✓ 원칙: 상품 중심, 자연스러운 조명, 미니멀한 배경
07. 접근성
색상 대비
요소 대비도 표준
본문 텍스트 7:1 (검정/흰색) AAA 준수
보조 텍스트 4.5:1 이상 AA 준수
버튼/링크 4.5:1 이상 AA 준수
모바일 최적화
  • 터치 타겟 최소 44px × 44px
  • 반응형 폰트 크기 (최소 14px)
  • 명확한 버튼과 링크
  • 충분한 탭 간격
  • 스크린 리더 지원
08. 글로벌 배포
뽀뿌리는 180개 이상의 국가에 배송하므로, 글로벌 사용자를 고려한 디자인이 필요합니다.
다국어 지원
  • 한글/영문 로고와 메뉴 제공
  • 한글/한자 병기로 전통성 표현
  • 텍스트 길이 변동에 대비한 유연한 레이아웃
  • 문화적 색감 배려 (예: 색상의 문화적 의미 고려)
국제 결제
전 세계 결제 방법 지원 (신용카드, PayPal, Apple Pay 등)
✓ 원칙: 포용적 디자인, 다양한 언어와 문화 존중
09. 지속적 개선
최신 경향 수용
  • AI 기반 개인화 (추천 시스템)
  • AR 피팅 (가상 한복 입어보기)
  • 다크 모드 지원
  • 모바일 앱 확대
  • 음성 검색 지원
성능 기준
지표 목표
페이지 로드 시간 < 2초
모바일 반응 시간 < 100ms
이미지 최적화 WebP 형식, 고압축
접근성 점수 > 95/100
